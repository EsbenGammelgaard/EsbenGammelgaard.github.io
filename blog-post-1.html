Connecting to my Local SQL Server from Python and Uploading Data

Published 4th May, 2021 5 min read

In this blog post, I would like to showcase how I work with python, the pyodbc, and the pandas module in order to access, manipulate, and upload data to my local SQL server. The SQL server was created in Microsoft SQL Server Management Studio 18 and is located on my personal pc. The data that is Uploaded to my SQL server originates from Kaggle.com and contains data regarding e-commerce data for an Indian webshop. The database consists of three files containing Lists of Orders, Order Details, and Sales Targets for a one-year period. From Kaggle.com the data is downloaded as .csv files. These three .csv files are uploaded to the SQL server, via Bulk upload, in temporary dataframes. The temporary dataframes contain all parameters stored in varchar(255). Afterwards the data is input into the permanent dataframes from the temporary. In this process, some of the dataformats are changed to save memory. Furthermore, data such as dates and numbers are change into the propper formats such as SMALLINT and DATE. The goal with uploading this data to the SQL server is to have it easily accessible in the following process where I plan to analyze and perhaps model some of the data.
